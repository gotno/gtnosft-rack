## Objective

Eliminate heap allocations from `LightState` storage and comparison by replacing `std::string hexColor` with `NVGcolor color`, and deferring hex string conversion to OSC message send time only.

## Context

The subscription fires every 30ms. Each tick, `LightState::update()` calls `rack::color::toHexString()` for every light just to compare whether the color changed—even in steady state with no changes. This allocates a `std::string` per light per tick (heap allocation). Additionally, `LightState` stores `std::string hexColor` as a member, meaning every cached light entry carries a heap allocation.

`NVGcolor` is a 16-byte POD struct (4 floats: r,g,b,a). Comparing two `NVGcolor` values directly (via `memcmp` or field comparison) is allocation-free and fast. Hex string conversion should only happen inside the `addMessage()` lambda body—when the message is actually serialized to the OSC stream.

## Tasks

- [ ] **Task 1: Replace `std::string hexColor` with `NVGcolor color` in `LightState`**

  Files: `src/osc/Bundler/ModuleLightsBundler.hpp`

  Changes:
  - In `LightState` constructor: change `hexColor(rack::color::toHexString(widget->color))` to `color(widget->color)`
  - Replace member `std::string hexColor;` with `NVGcolor color;`

- [ ] **Task 2: Update `LightState::update()` to compare `NVGcolor` directly**

  Files: `src/osc/Bundler/ModuleLightsBundler.hpp`

  Changes:
  - Remove `std::string currentHexColor = rack::color::toHexString(widget->color);`
  - Compare using `memcmp(&color, &widget->color, sizeof(NVGcolor)) == 0` instead of `hexColor == currentHexColor`
  - Update assignment: `color = widget->color;` instead of `hexColor = currentHexColor;`

- [ ] **Task 3: Move `toHexString()` call into `addMessage()` lambda body**

  Files: `src/osc/Bundler/ModuleLightsBundler.cpp`

  Changes:
  - Keep the existing `[moduleId, state]` lambda capture unchanged
  - Inside the lambda body, replace `state.hexColor.c_str()` with `rack::color::toHexString(state.color).c_str()`
  - (The `hexColor` member is removed in Task 1; `state.color` is the `NVGcolor` added there)
